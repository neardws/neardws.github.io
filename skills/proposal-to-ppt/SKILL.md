# 基金申报书转答辩PPT

将项目申报书内容按答辩PPT模板结构重写，输出每页PPT的文字内容。

## 工作流程

### 第一步：提取文件信息

```bash
# 查看PDF基本信息
pdfinfo <申报书.pdf>
pdfinfo <PPT模板.pdf>

# 提取文本内容
pdftotext <申报书.pdf> - | head -200  # 先看前200行
pdftotext <PPT模板.pdf> -              # 提取完整PPT结构
```

### 第二步：分析PPT模板结构

从模板PDF中识别：
- 总页数和每页标题
- 汇报提纲（通常4个部分）
- 每个部分包含的页面

典型答辩PPT结构（18页）：
1. 封面页
2. 汇报提纲
3-6. 项目背景与总体思路（4页）
7. 提纲过渡页
8-11. 研究内容与技术路线（4页）
12. 提纲过渡页
13-14. 研究团队与工作基础（2页）
15. 提纲过渡页
16-17. 研究计划与考核指标（2页）
18. 致谢页

### 第三步：提取申报书关键内容

从申报书中提取以下部分：

```bash
# 分段提取
pdftotext <申报书.pdf> - | sed -n '1,100p'    # 封面和目录
pdftotext <申报书.pdf> - | grep -A 50 "研究背景"
pdftotext <申报书.pdf> - | grep -A 100 "研究目标"
pdftotext <申报书.pdf> - | grep -A 100 "研究内容"
pdftotext <申报书.pdf> - | grep -A 50 "人才条件"
pdftotext <申报书.pdf> - | grep -A 50 "研究进度"
```

需要提取的内容：
- 项目名称、负责人、申报单位
- 研究背景与需求分析
- 国内外研究现状
- 研究目标与研究内容
- 技术路线与关键技术
- 团队信息与研究基础
- 进度安排与考核指标

### 第四步：内容重写原则

**格式要求：**
- 使用列表格式而非表格（避免渲染截断）
- 每条内容完整，不使用省略号
- 标题简洁，正文精炼

**每页内容结构：**
```markdown
## 第N页：页面标题

**要点一**
- 具体内容

**要点二**
- 具体内容

**➤ 本页结论/要点**
```

### 第五步：各页内容要点

**封面页：** 项目名称、负责人、申报单位、汇报日期

**项目背景（3-4页）：**
- 第一页：面临的挑战（问题是什么）
- 第二页：国内外研究现状与差距
- 第三页：核心技术难题（凝练为2-4个）
- 第四页：总体思路（挑战→问题→内容→目标）

**研究内容（3-4页）：**
- 第一页：总体技术路线
- 后续每页：一个研究内容（难点→方法→预期成果）

**研究团队（1-2页）：**
- 负责人信息
- 团队成员与分工
- 研究基础与工作条件

**进度与指标（2页）：**
- 进度安排：按季度列出任务
- 考核指标：报告、软件、论文、专利

### 第六步：配图生成

使用 nano-banana-pro 为关键页面生成配图：

```bash
uv run /path/to/nano-banana-pro/scripts/generate_image.py \
  --prompt "图片描述，NO TEXT" \
  --filename "output.png" \
  --resolution 1K
```

**配图建议：**
- 项目背景页：展示研究领域的挑战场景
- 技术路线页：框架图或流程图
- 要求无文字（NO TEXT）便于后期添加中文标注

### 第七步：输出格式

输出为Markdown文件，每页用二级标题分隔：

```markdown
# 答辩PPT内容

## 第1页：封面
...

## 第2页：汇报提纲
...
```

保存到与源文件同目录：
```bash
/path/to/presentation/ppt_content_output.md
```

## 注意事项

1. **内容一致性**：总体思路中的"挑战-问题-内容"要与后续研究内容页一一对应
2. **文字精炼**：每条发展趋势/差距控制在一行内
3. **避免表格**：使用列表格式防止渲染截断
4. **目标简洁**：研究目标控制在两行以内
5. **配图无字**：生成图片时强调NO TEXT，便于后期添加中文

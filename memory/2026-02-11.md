## Coding Agent 统一配置调研

### 背景
Neil 想要统一配置 5 个 coding agent 的 LLM API key：
- Droid (Factory AI)
- Claude Code (Anthropic)
- Codex (OpenAI)
- Gemini CLI (Google)
- Opencode (MiniMax)

### 调研结论

所有 5 个 agent 都支持 API Key 认证：

| Agent | 认证方式 | 配置位置 |
|-------|----------|----------|
| Claude Code | `ANTHROPIC_AUTH_TOKEN` | 环境变量 |
| Droid | `DROID_API_KEY` | 环境变量 |
| Codex | `OPENAI_API_KEY` | `codex login --with-api-key` 或环境变量 |
| Gemini CLI | `GEMINI_API_KEY` | 环境变量或 `~/.gemini/.env` |
| Opencode | `apiKey` | `~/.config/opencode/config.json` |

### 推荐方案

**LiteLLM Gateway + 统一环境变量**

```yaml
# litellm_config.yaml
model_list:
  - model_name: claude-opus-4-6
    litellm_params:
      model: anthropic/claude-opus-4-6
      api_key: os.environ/ANTHROPIC_API_KEY
  - model_name: gpt-5.3-codex
    litellm_params:
      model: openai/gpt-5.3-codex
      api_key: os.environ/OPENAI_API_KEY
  - model_name: gemini-2.5-pro
    litellm_params:
      model: gemini/gemini-2.5-pro
      api_key: os.environ/GEMINI_API_KEY
  - model_name: MiniMax-M2.1
    litellm_params:
      model: openai/MiniMax-M2.1
      api_base: https://api.minimaxi.com/v1
      api_key: os.environ/MINIMAX_API_KEY
```

**各 Agent 指向 Gateway:**

```bash
export LITELLM_URL="http://localhost:4000"
export LITELLM_KEY="sk-your-master-key"

# Claude Code
export ANTHROPIC_BASE_URL="${LITELLM_URL}/anthropic"
export ANTHROPIC_AUTH_TOKEN="${LITELLM_KEY}"

# Droid
export DROID_API_KEY="${LITELLM_KEY}"

# Codex
export OPENAI_BASE_URL="${LITELLM_URL}"
export OPENAI_API_KEY="${LITELLM_KEY}"

# Gemini CLI
export GEMINI_API_KEY="${LITELLM_KEY}"
```
